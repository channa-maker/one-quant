# 第一部分 · 总览与架构

## 1. 项目总览

### 1.1 一句话定义

ONE量化 是一套面向机构内部、覆盖**加密货币(现货/合约/期权)与美股/美期权**的全自动智能量化交易系统,以 **LLM 定性分析 + ML 定量预测**双 AI 引擎驱动选股选币与策略决策,采集**全量真实数据投喂 AI 实现自我进化**,通过**四层硬风控**保障资金安全,具备**专业级多屏盯盘**,实现 **7×24 小时无人值守自动交易**。

### 1.2 核心能力清单

| 能力域 | 说明 |
|--------|------|
| 一站式多市场 | 统一抽象层桥接 IBKR(美股/美期权)+ 币安/OKX(加密现货/合约)+ Deribit(加密期权) |
| AI 分析 | LLM 行情解读、新闻情绪、宏观研判、自动中文研报 |

## 2. 产品定位与设计铁律

### 2.1 定位

- **机构内部平台**:服务自营/私募/工作室,多策略多账户,强风控强审计,不做多租户计费与对外 KYC。
- **资金安全 > 收益**:任何收益逻辑先过风控;风控拥有绝对否决权。
- **可解释**:每笔交易都能回答"为什么买、为什么这个量、为什么这个价"。

### 2.2 设计铁律(不可妥协)

1. **风控阈值硬编码**:最大回撤、最大杠杆、单仓上限是代码常量,不读 `.env`、不读 DB。
2. **不可变审计**:风控决策、订单状态变更追加写入,只增不改。
3. **故障即停**:风控代码异常必须 ERROR 日志 + 触发熔断,**禁止 `except: pass`**。
4. **行情永远主网**:即使测试网下单,行情取主网。
5. **幂等下单**:所有订单带 `clientOrderId`(UUIDv4),重试不重复。
6. **测试网/主网物理隔离**:适配器初始化即校验。
7. **AI 无否决权**:AI 只产建议/信号,永远绕不过风控。
8. **小文件高内聚**:单文件 200–400 行典型,800 行上限。
9. **中文优先**:所有面向人的输出默认中文。
10. **先影子后实盘**:任何新逻辑(策略/模型/执行算法)先影子→灰度→主用。

## 3. 系统总体架构

### 3.1 三路径分层(热 / 温 / 冷)

```
┌─────────────────────────────────────────────────────────────────┐
│                          接入层 / 前端                            │
│  Web 交易终端  ·  多屏盯盘工作站  ·  移动端  ·  Grafana 大盘      │
└───────────────┬─────────────────────────────────┬───────────────┘
                │ HTTPS / WebSocket                │
┌───────────────▼─────────────────────────────────▼───────────────┐
│                        API 网关层(FastAPI)                       │
│   认证鉴权 · RBAC · 限流 · 幂等校验 · 审计中间件 · WS Hub          │
└───────────────┬─────────────────────────────────────────────────┘
                │ EventBus(Redis Pub/Sub + Streams)
┌───────────────▼─────────────────────────────────────────────────┐
│  热路径(微秒~毫秒)   行情接入 → 策略引擎 → 风控引擎 → 执行引擎    │
│  温路径(秒~分钟)     调度器 · AI 智能体 · 选股选币 · 持仓管理     │
│  冷路径(离线批处理)  数据湖 · 因子计算 · 模型训练 · 回测引擎      │
└───────────────┬─────────────────────────────────────────────────┘
                ▼
   存储层:TimescaleDB(时序)· PostgreSQL(交易/审计)·
          ClickHouse(分析)· DuckDB/Parquet(数据湖)· Redis(缓存/总线)· MinIO(对象)
```

### 3.2 进程模型

| 进程 | 职责 | 运行时 |
|------|------|--------|
| `one-api` | HTTP/WS、鉴权、查询 | Python/FastAPI |
| `one-runner` | 策略主循环(单 asyncio 事件循环) | Python |
| `one-marketgw` | 行情网关、低延迟分发 | Rust(可选)/Python |
| `one-risk` | 风控守护(L3 后台轮询) | Python |
| `one-ai` | AI 智能体调度 | Python |
| `one-trainer` | ML 离线训练管线 | Python/PyTorch |
| `one-collector` | 全量数据采集落湖 | Python |
| `one-watchdog` | 看门狗、全进程心跳与自愈 | Python |

> 进程间**只通过 EventBus 通信**,绝不直接调用。消息为 JSON 信封 `{channel, schema_version, seq, ts_ns, trace_id, data}`。测试用 `InMemoryEventBus` 免 Redis。

### 3.3 核心数据流(端到端)

```
交易所/券商行情 → 行情网关(归一化) → EventBus(market.*)
  → 策略引擎 on_ticker/on_kline/on_orderbook → Signal
  → 风控网关 check()(L1→L2→L3→L4) → APPROVE|REJECT|REDUCE|FLATTEN
  → 执行引擎 OMS.submit() → 算法拆单 → 适配器 → 交易所/券商 API
  → 成交回报 → 持仓更新 → EventBus(fill.*) → 前端/审计/数据湖/AI
```

## 4. 底层架构深化

> 地基铁律:确定性、可回放、可观测、低延迟、可演进。

### 4.1 事件溯源 + CQRS

- **事件溯源**:所有状态变更以不可变事件流记录;任意时刻状态 = 事件重放结果。
- **CQRS 读写分离**:写模型(命令)与读模型(查询/盯盘投影)分离,盯盘大盘不拖累交易主路径。
- **确定性回放**:同一事件流 + 同一代码 = 同一结果,实盘问题离线 100% 复现(排障与回测一致性基石)。

### 4.2 消息契约与总线

- 统一信封:`{channel, schema_version, seq, ts_ns, trace_id, data}`。
- `seq` 单调递增 + `trace_id` 全链路 + `ts_ns` 纳秒戳。
- 关键通道(订单/成交/风控)持久化(Redis Streams / Kafka 可选),保证不丢、可重放;高频行情走 Pub/Sub(可丢、要快)。

### 4.3 时间与时钟

- NTP/PTP 校时,全系统纳秒时间戳;交易所时间与本地偏差监控。
- 严格区分**事件时间 vs 处理时间**,回测/实盘都用事件时间防未来函数。
- 延迟预算分段打点:行情接收→解码→策略→风控→下单,每段 P99 看板。

### 4.4 低延迟与性能分层

- 热路径:延迟敏感策略(做市/订单流)可下沉 Rust/C++ 或独立高优先级进程;对象池、零拷贝、避免 GC 抖动。
- 温路径:Python asyncio + uvloop。
- 背压流控:每级队列有界,满了走明确策略(丢最旧/降级/告警),绝不无限堆积。
- 数据本地化:热数据内存/Redis,温数据 TimescaleDB,冷数据 ClickHouse/Parquet。

### 4.5 容错与可演进

- 幂等贯穿(下单/消费/恢复);本地 vs 交易所定时对账,不一致自动修正 + 告警。
- 故障域隔离:单策略/单适配器崩溃不扩散;风控异常立即熔断。
- 优雅降级:AI 不可用→退回规则策略;次要数据源断→主源继续;盯盘断→交易不受影响。
- 模块化单体起步,按需拆服务(微服务路径预留,不过早微服务化);核心域(风控/执行/策略契约)稳定,外围(因子/AI/前端)高频迭代。

# 第二部分 · 数据与接入

## 6. 一站式多市场接入

> 工程真相:**无单一券商一账户覆盖"美股+美期权+全球加密现货/合约/期权"**。一站式靠**统一抽象层**实现:对上层呈现一个统一接口,底层桥接多家,既能统一操作又不被单一厂商绑死。

### 6.1 统一券商抽象层

```
        策略 / AI / 盯盘(只认统一接口)
                   │
        ┌──────────▼──────────┐
        │  UnifiedBroker 接口  │  下单/撤单/持仓/行情/资金 统一语义
        └──────────┬──────────┘
   ┌────────┬──────┼──────┬────────┬─────────┐
   ▼        ▼      ▼      ▼        ▼         ▼
 IBKR    Alpaca  Binance OKX    Deribit   (预留新券商插件位)
(美股+   /tasty  (现货+  (现货+  (加密期权
 美期权) (美股期权)合约+   合约+   主力流动性)
                 加密期权)期权)
```

- **统一语义**:各家"股票/期权/永续/现货"映射到内部统一 `Instrument` + `Order`,屏蔽委托类型/保证金/合约乘数/结算差异。
- **能力声明**:每适配器声明支持的市场/订单类型/特性,上层按能力路由,不支持优雅降级。
- **智能路由**:同标的多渠道可成交时按价格/流动性/费率/延迟择优。
- **唯一入口**:`BrokerPool`/`ExchangePool` 惰性创建、用后释放,禁止直接 new。

### 6.2 接入方案(截至 2026-01,待联网核实)

| 市场 | 候选 | 说明 |
|------|------|------|
| 美股 + 美期权 | **IBKR**(主) | 覆盖最广,一账户多市场,机构首选。待核实当前 API 限额/期权链接口 |
| 美股 + 美期权 | Alpaca / tastytrade / Tradier(备) | API 友好;加密覆盖有限。待核实期权全链支持 |
| 加密 现货/合约 | Binance / OKX / Bybit | 7×24,L2/tick 全,限额宽 |
| 加密 期权 | **Deribit**(主)/ Binance / OKX | Deribit 为加密期权事实标准。待核实合约与费率 |

> 落地顺序:**先加密(数据全、7×24、无牌照门槛)跑通 → 再接 IBKR 拿下美股+美期权**。

### 6.3 一站式体验与调度

- 前端呈现"一个系统":标的搜索跨市场统一(`AAPL` / `BTC-PERP` / `BTC-30JUN-CALL` 都能下单)。
- 统一持仓/资金视图:跨券商跨市场净敞口、保证金、组合 Greeks、总盈亏一屏看全。
- 资金调拨:跨券商需人工划转(合规红线),系统只给建议。
- 调度器内置各市场交易日历 + 节假日 + 夏令时:加密 7×24;美股含盘前/盘中/盘后,自动启停对应策略。

## 7. 本地全量数据采集与数据湖

> 核心:**采集所有真实数据落本地,AI 拿本地全量数据做分析/对比/进化**。这是数据主权,也是订单流/聪明钱/ML/LLM 的燃料。

### 7.1 采集清单(全量真实数据)

| 类别 | 具体 | 频率 | 用途 |
|------|------|------|------|
| 逐笔成交 Tick | 价/量/方向 | 实时 | 订单流、CVD、回测一致性 |
| 盘口 L2/增量 | 多档挂单增删改 | 实时 | DOM、热力图、失衡因子 |
| 逐单 MBO(可得时) | Market-By-Order | 实时 | 冰山/扫单/吸收识别 |
| K 线多周期 | 1s/1m/5m/1h/1d | 聚合 | 指标、展示 |
| 资金费率/OI/多空比 | 衍生品结构 | 1m | 加密结构因子 |
| 期权链/Greeks/IV | 全链 + 希腊值 | 1m | 期权策略、波动率面 |
| 清算/强平 | 爆仓流 | 实时 | 清算热力、反转 |
| 链上 | 交易所流入流出、巨鲸、稳定币 | 分钟/区块 | 聪明钱(加密) |
| 新闻/公告 | 财经新闻、交易所公告、SEC 文件 | 事件 | LLM 情绪/事件面 |
| 社媒情绪 | X/Reddit/电报(合规范围) | 实时 | 情绪因子 |
| 宏观日历 | CPI/非农/FOMC/财报 | 事件 | 宏观研判 |
| **自身交易数据** | 订单/成交/滑点/盈亏/决策留痕 | 实时 | **自进化反馈样本(最重要)** |

### 7.2 数据湖架构(Medallion 三层)

```
采集层 Collectors(多源冗余、断线重连、去重校验)
   ▼ 原始落盘(不丢任何真实数据)
Bronze 青铜:原始未加工 → Parquet / ClickHouse 原始表(只增不改、可重放)
   ▼
Silver 白银:清洗对齐(去重、事件时间对齐、质检、归一化)→ TimescaleDB(热)+ ClickHouse(温)
   ▼
Gold 黄金:特征/因子(聚合、特征商店离线+在线一致)→ Feature Store(Parquet + Redis 在线)
   ▼
统一查询接口(供 AI / 回测 / 研究 / 盯盘)
```

- **不丢任何真实数据**:Bronze 原始永久保留,任意时刻从头重放。
- **数据质检门**:乱序丢弃+告警、跳变标记、缺口回补、重复去重。
- **数据血缘**:因子→Silver→Bronze→数据源全程可追溯。
- **存储规划**:tick/L2 体量巨大,列存压缩(ZSTD)+ 冷热分层 + 对象存储。

### 7.3 投喂 AI 的统一接口

> AI 不直连交易所,**只吃本地数据湖**,保证一致、可复现、可回放。

- **特征商店**:离线训练与在线推理特征同源,杜绝训练-服务偏差。
- **AI 数据 API**:ML 训练读 Gold 因子+标签(时间对齐防未来函数);LLM 分析走 RAG 检索本地新闻/研报/历史决策 + 结构化行情摘要;在线推理 Redis 毫秒取数。
- **机构记忆**:历史决策/复盘/事故/研报入向量库,LLM 决策前检索"以前类似情况怎么处理",积累机构经验。

### 7.4 数据合规
遵守各交易所/数据商 API 条款与再分发限制(美股 L2 通常付费且禁再分发,内部自用合规);社媒采集守平台条款;本地加密存储 + 访问审计。

### 7.5 数据完整性 —— 幸存者偏差 + Point-in-Time〔G2 · 🔴〕
> 地基级,阶段一必须埋好,后补代价极大。
- **保留已退市/已下架标的全历史**:加密死币、美股退市股不删,回测标的池按历史时点动态构建(当时在市全集),消除幸存者偏差。
- **Point-in-Time(PIT)数据**:基本面/指数成分/评级用"当时已知"版本快照,不用修订后数据,防未来函数。
- **数据版本/快照**:关键数据集打时间快照,保证回测可复现。

### 7.6 标的主数据 / 符号映射(Symbology)〔G10 · 🟠〕
> 地基级,跨市场聚合的前提。
- **Instrument Master**(标的主数据库):统一内部标的 ID,映射各交易所符号(BTCUSDT / BTC-USD / XBTUSD)。
- **符号变更历史**:美股 ticker 变更、加密改名留痕;ISIN/FIGI/CUSIP 外部标识映射。
- **新标的上线校验流程(Onboarding)**〔G19〕:流动性/数据质量/风控参数/合规四项校验通过才纳入交易。

## 8. 策略引擎

### 8.1 策略基类契约

所有策略继承 `Strategy` ABC(`strategy/contracts.py`),禁止绕过:

```python
class Strategy(ABC):
    """策略基类。子类实现行情回调,返回信号列表。"""
    @abstractmethod
    def on_ticker(self, ticker: Ticker) -> list[Signal]: ...
    @abstractmethod
    def on_kline(self, kline: Kline) -> list[Signal]: ...
    def on_orderbook(self, ob: OrderBook) -> list[Signal]: return []
    def on_option_quote(self, q: OptionQuote) -> list[Signal]: return []
    def on_fill(self, fill: Fill) -> None: ...
    def on_recover(self, state: PositionState) -> None: ...  # 重启状态恢复
```

**强制规范**:每方法完整类型标注;策略类必须 docstring(核心思想/参数/不同市场环境预期);因子命名 `{类别}_{名称}_{窗口}`;NaN 显式处理禁止静默传播;新策略默认禁用,回测+评审通过才启用。

### 8.2 内置策略库

| 均值回归 | RSI 反转、布林带、配对交易 | 现货/合约 |
| 做市 | 双边挂单、库存偏斜调整 | 现货/合约 |
| 套利 | 跨所价差、资金费率、期现、三角 | 现货/合约 |

### 8.3 出场架构(两层)

> 教训:出场逻辑历史出过致命 bug(重启后入场价 key 丢失致出场死循环),严格设计。

- **第一层 固定出场**:止损/止盈/移动止损,参数化可回测。
- **第二层 ExitBrain(数据驱动影子层)**:只读旁路,基于持仓特征预测最优出场,与固定出场 A/B,灰度成熟再切主。
- **状态恢复**:重启从持仓表重建 SL/TP 与**所有入场价 key**(回归测试覆盖)。

### 8.4 回测要求

每策略须有 `tests/test_strategy_{name}.py`:锚定交易测试、无未来函数测试、空数据测试、成本扣除测试。回测输出必含:年化收益、最大回撤、夏普、卡玛、胜率、盈亏比、换手率;必报样本内 vs 样本外。

## 9. 高级策略体系:订单流 / 聪明钱 / 微观结构

> 机构 alpha 核心来源。基于逐笔成交与盘口微观结构,而非滞后 K 线指标。**依赖 tick + L2 数据(第 7 章)。**

### 9.1 订单流(Order Flow)族

| 信号 | 含义 | 用法 |
|------|------|------|
| Delta / CVD | 主动买-卖累计差 | 价格新高但 CVD 不创新高 = 背离,警示反转 |
| 失衡 Imbalance | 同价位 Bid/Ask 成交悬殊 | 连续买方失衡 = 强势吸筹 |
| 吸收 Absorption | 大量主动单被吃而价不动 | 主力吸筹/派发 |
| 扫单 Sweep | 瞬间扫多档流动性 | 动量信号 |
| 冰山单 Iceberg | 隐藏大额挂单(不断补) | 主力支撑/压力 |
| 盘口失衡 OBI | 买卖挂单量比 | 短期方向因子 |
| 流动性墙 | 盘口异常大挂单(热力图) | 磁吸价位/假墙 |

落地组件:Footprint、DOM、CVD、Heatmap(第 18 章),既可视化也是因子源。

**反幌骗(Spoofing)防护**〔G13 · 🟠〕:订单流易被假大单诱导。过滤手段:撤单率/挂单存活时间过滤、瞬时撤单识别、假墙剔除;只采信真实成交印证的盘口信号,挂单信号需成交确认。

### 9.2 聪明钱 / SMC(Smart Money Concepts)族

| 概念 | 中文 | 建模 |
|------|------|------|
| BOS / CHoCH | 市场结构破坏/转换 | 自动识别高低点结构、趋势转折 |
| Order Block | 订单块 | 机构挂单密集区,回踩支撑/压力 |
| Fair Value Gap | 公允价值缺口 | 失衡缺口,价格倾向回补 |
| Liquidity Pool | 流动性池 | 等高/等低点止损聚集区 |
| Liquidity Grab | 流动性猎杀 | 假突破扫止损后反转 |
| Premium/Discount | 溢价/折价区 | 斐波那契均衡线判多空 |
| Smart Money Index | 聪明钱指数(聪明线) | 早盘散户情绪段 vs 尾盘机构段累计差 |

> 「聪明线」两种都实现:经典 Smart Money Index + SMC 结构线(自动画 BOS/CHoCH/OB/FVG),作为独立因子 + 盯盘叠加层。

### 9.3 量价结构族
成交量分布 VPVR(POC/价值区/HVN-LVN)· 市场轮廓 TPO · VWAP 家族(锚定 VWAP/标准差带/机构成本线)· 多周期 SMC/订单流共振。

### 9.4 加密专属结构
链上聪明钱(交易所净流入流出、巨鲸、稳定币)· 衍生品结构(资金费率极值、OI 变化、多空比、清算热力图)· 期权结构(Max Pain、Gamma 暴露 GEX、PCR、IV 偏斜)。

### 9.5 策略融合层
订单流(微观)+ SMC(中观)+ 因子/ML(统计)+ LLM(消息面)四层共振才高置信;ML 作融合器(订单流/SMC/量价作特征 → 综合置信度);LLM 作复核器(高置信信号做消息面一票否决);融合输出仍是 `Signal`,**必过四层风控**。

## 12. AI 数据→分析→对比→进化闭环

### 12.1 闭环全景

```
本地全量数据(第7章)
 ① 分析:LLM 定性 + ML 定量(订单流/SMC/因子/情绪)
 ② 对比:多策略/多模型/多市场横向对比 + 历史同类情形比对(机构记忆)
 ③ 假设:LLM 基于对比提出可检验假设(新因子/参数/组合)
 ④ 验证:自动回测 + 样本外 + 影子运行,真实数据校验
 ⑤ 进化:达标灰度→晋升(冠军挑战者);衰减降权退役
 ⑥ 反馈:实盘成交/滑点/盈亏回流数据湖,成为下一轮样本
 └──────────────── 回到 ① ────────────────┘
```

### 12.2 「对比」是进化关键(易忽略)
跨策略对比(找市场环境-策略适配)· 预测 vs 实际(误差回流校准)· 回测 vs 实盘(偏差>0.05% 告警)· 历史情形对比(向量检索相似形态类比)· AI 自评(LLM 复盘对错,生成中文进化报告)。

### 12.3 进化安全闸(防 AI 翻车)
任何 AI 改动必经回测+影子+灰度三关,不直接上实盘;进化产物仍是 Signal 必过风控,硬阈值 AI 改不动;进化全审计(依据什么数据、对比什么、为什么);防过拟合(样本外+IC/ICIR 衰减+多周期稳健)。

### 12.5 AI 信号推荐与评分系统(横跨 AI 与盯盘)

> 把全量数据分析结果凝练成**校准过的评分 + 可解释信号卡**,智能推送到盯盘工作站,辅助人机协同并闭环自校准。详见独立文档《ONE量化_AI信号推荐与评分系统.md》。

**设计哲学**:评分必须校准(85 分 = 历史真实胜率约 85%)· 高分只来自多源共振(单源封顶中分)· 多维信号卡(非单数字)· 反噪音(冷却/去重/regime/分级)· 闭环对账自校准 · 中文可解释 · 信号只建议不越权(自动单仍必过四层风控)。

**评分模型**:
```
综合分 Score = Calibrate( Σ wᵢ · sᵢ · dᵢ ),范围 0–100
  sᵢ 证据强度(0–1)· dᵢ 方向(+1多/-1空/0中)· wᵢ 权重(历史 IC/胜率动态学习)
  Calibrate 校准函数(Isotonic/Platt,分市场,滚动再校准,把原始分映射到真实胜率)
```
- 证据源:订单流(CVD/失衡/吸收/扫单/OBI)· SMC(BOS/CHoCH/OB/FVG/猎杀)· 量价(VPVR/VWAP/TPO)· ML 打分 · LLM(情绪/宏观/事件)· 加密结构(费率/OI/清算/GEX)· 链上(流向/巨鲸)。
- 共振加成(≥3 独立源同向加成,相关源不重复计分)· 单源封顶(逼高分多源)· 冲突衰减(矛盾时向中性收敛+标注分歧)。

**信号分级**:S(≥85,声音+弹卡+移动端)· A(70–84,弹卡无声)· B(55–69,进列表)· C(<55,仅记录)。

**信号卡(多维)**:标的/方向/综合分/置信区间/级别/时间窗(短炒·日内·波段)/风险/建议止损/盈亏比/中文理由/证据明细(各源贡献)/历史同类胜率(向量检索相似形态)/信号ID。

**盯盘推送与反噪音**:信号面板常驻滚动 · 图表叠加分数气泡 · 仅 S 级声音/语音播报 · 移动端推 S 级+自选 · 冷却期/去重合并/市场状态感知(震荡抑制趋势信号)/升级才再推/流控上限 · 信号卡支持一键跟单(过风控+二次确认)/加入观察/👍👎反馈/查看复盘。

**闭环自校准**:每个推送信号自动跟踪后续真实走势→打标(赢/输/幅度)→回流数据湖→校准曲线再拟合+证据源权重再学习(分市场、样本外验证防过拟合)→下一批更准;👍/👎 作高权重人工标注;信号绩效看板(各级命中率/证据贡献/校准漂移,中文 Grafana)。

**与自动交易边界**:信号只产建议;自动模式仅 S 级(阈值可配)可触发下单且必过四层风控,硬阈值不可越;信号→订单全程审计可回溯到触发信号卡与证据明细。

| ①因子发现 | FactorDiscoverer(LLM+遗传) | IC/ICIR 阈值 |
| ②策略生成 | 因子组合/参数搜索/AutoML | — |
| ③回测 | 自动+样本外+无未来函数 | 夏普/回撤门槛 |
| ④风险评估 | 压力测试+相关性(与现有低相关) | 风控签字 |
| ⑤影子 | 只读跟单对比预测 | 一致性达标 |
| ⑥灰度 | 小资金真实下单 | 实盘指标达标 |
| ⑦全量 | 提升资金权重 | 人工+风控双签 |
| ⑧监控 | 实时表现追踪 | — |
| ⑨衰减 | alpha 衰减/过拟合复发检测 | 自动降权 |
| ⑩退役 | 自动降权/下线归档 | — |

### 13.2 冠军-挑战者机制
每槽位一个"冠军"在跑,多个"挑战者"影子并行;挑战者真实行情影子运行持续 PK(夏普/回撤/稳定);稳定超越自动晋升(灰度→全量),冠军降级/退役;全程审计可回溯换代原因。

### 13.3 自动因子挖掘
遗传规划/符号回归(自动组合算子)· AutoML 贝叶斯搜参(Optuna)· LLM 假设生成(读市场+文献提因子假设)· 因子去冗余(与已有低相关,正交化)。

### 13.4 模型在线学习与再训练
滚动再训练(日/周用新数据)· 概念漂移检测(分布漂移/残差触发再训练)· 在线学习(可选,谨慎,需影子)· 模型版本灰度+一键回滚。

### 13.5 系统级自我升级
配置热更新(参数/扫描规则/告警阈值热加载,风控硬阈值除外)· 策略热插拔(插件加载/卸载不重启 runner)· 金丝雀部署 · 自动回滚(关键指标劣化回退上一版)。

### 13.6 模型风险管理(MRM)〔G7 · 🟠〕
> 机构跑 ML/LLM 决策资金的治理底线。模型注册表升级为 MRM 平台:
- **模型清单(Inventory)**:每个模型登记用途/输入数据/责任人/版本/状态。
- **独立验证**:验证团队与开发分离,上线前出验证报告(性能/稳健/偏差)。
- **模型卡(Model Card)**:假设、局限、适用边界、失效条件中文留档。
- **生命周期审批**:上线/重大变更/退役需审批;血缘可追溯(数据→特征→模型→信号→订单)。
- **持续监控**:性能衰减、漂移、异常预测触发复核或自动降权下线。

### 13.7 长稳一致性测试(Soak Test)〔G20 · 🟡〕
新策略影子运行须跑足够长周期(建议 ≥1 月)验证一致性与稳定性,而非短测即上;影子-实盘偏差持续达标才进灰度。

## 14. 板块模块:现货 / 合约 / 期权

### 14.1 现货
交易所:币安/OKX 现货、美股(IBKR)。能力:限价/市价/冰山、网格、定投、再平衡。防假成交/部分成交。

### 14.2 合约
币安 U/币本位、OKX 永续。能力:多空双向、杠杆控制(硬上限)、资金费率监控、强平价计算、自动减仓预警。风控重点:**杠杆上限硬编码**,临近强平自动 REDUCE/FLATTEN。

### 14.3 期权
币安期权 / Deribit、美期权(IBKR)。能力:期权链浏览、实时 Greeks 与 IV 曲面、组合策略(垂直/跨式/宽跨式/铁鹰/日历/领口)、**组合层 Greeks 风控**(总 Delta/Gamma/Vega/Theta 限额)、卖方保证金监控、被行权预警。IV 套利模型 + 盘口过滤;临近到期自动展期/平仓提示。

### 14.4 跨板块统一抽象
统一 `Instrument`(`{market, symbol, type: spot|futures|option}`)· `ExchangePool` 唯一获取适配器入口 · 统一持仓视图(跨市场净敞口、保证金、组合 Greeks 聚合)。

### 14.5 公司行为(Corporate Actions)处理〔G1 · 🔴 美股关键〕
> 漏了会持仓数量算错、回测用未复权数据=假信号、期权乘数错乱。接 IBKR 前必补。
- **公司行为日历 + 自动处理引擎**:分红、拆股/合股、配股、并购、特别股息。
- **历史价格复权**:除权自动调整历史 K 线(前复权/后复权可选)与持仓成本。
- **期权调整**:拆股导致的非标准合约(调整后乘数/行权价)正确建模。
- **退市/下架**:预警 + 强制平仓流程;退市标的历史保留(配合 G2 防幸存者偏差)。

### 14.6 美股市场机制约束〔G3 · 🔴〕
> 美股硬规则,不处理会被券商拒单或违规。美股适配器内置校验,风控 L1 增美股专属规则集(预交易拦截)。
- **PDT(Pattern Day Trader)**:<$25k 账户 5 日内 ≤3 次日内往返,超限拦截。
- **Reg-T 保证金**:初始 50%/维持 25%(与加密杠杆体系不同)。
- **SSR(卖空限制)**:个股跌 10% 触发,卖空只能高于最优买价。
- **借券定位(Locate)**:卖空前校验可借券源。
- **LULD / 市场级熔断**:个股限价带 + 全市场熔断停盘识别与暂停下单。
- **T+1 结算 / 现金账户**:未交收资金限制。

## 15. 四层风控引擎

> 风控拥有**绝对否决权**,任何策略/AI/人工不能绕过;紧急绕过须先 `halt_all()`。

### 15.1 四层结构

| 层 | 名称 | 检查项 | 延迟 |
|----|------|--------|------|
| L1 | 静态限额 | 白名单、单笔最小/最大名义、可交易性 | <1ms |
| L2 | 实时敞口 | 最大敞口、下单频率、价格偏离、杠杆上限 | <2ms |
| L3 | 后台回撤 | 最大回撤 kill switch(每 1s 轮询)、保证金率 | 1s |
| L4 | 熔断器 | 开/关/半开三态,连续异常触发全局熔断 | 事件 |

### 15.2 四态决策(铁律)
每次检查必返回四态之一,禁止二元:`APPROVE 批准 / REJECT 拒绝 / REDUCE 减仓后批准 / FLATTEN 强制平仓`。

### 15.3 硬约束
1. 风控阈值是代码常量(不读 `.env`/DB)。
2. 禁止 `except Exception: pass`(必 ERROR 日志 + 重抛或置熔断)。
3. 决策不可变审计(纳秒戳、策略ID、订单ID、决策、触发规则、状态快照,只增不改)。
4. 风控可组合(新规则实现 `RiskRule` 协议追加规则链)。

### 15.4 组合层 + 全局闸(补强)
预交易 + 盘中 + 盘后三段风控;组合 VaR/CVaR、相关性矩阵、行业/板块暴露限额;同标的跨策略净敞口合并计算(防同向叠加爆仓);**全局日内最大亏损硬上限触发 `halt_all()`**;黑天鹅预案(交易所宕机/闪崩/稳定币脱锚/插针应对剧本)。
**组合优化器**〔G14 · 🟡〕:均值-方差 / 风险平价 / Black-Litterman 在策略与标的间分配权重 + 波动率目标,作为资金分配引擎的优化内核。

### 15.5 测试要求
每规则:正常通过/边界拒绝/边缘情况三测;新规则需混沌工程测试(订单洪流/价格跳空/交易所断连);风控引擎覆盖率 **≥ 95%**;编辑风控代码前跑全量测试+检查熔断器+确认无实盘策略运行。

### 15.6 压力测试场景库(历史危机回放)〔G11 · 🟠〕
> 混沌工程测随机故障,这里用真实极端行情测抗压。
- **危机场景库**:2020 新冠闪崩、2022 LUNA 崩盘、FTX 暴雷、美股熔断日、312/519 等。
- **定期回放**:用真实极端 tick 回放,测系统稳定性 + 策略抗压 + 风控触发是否及时。
- **极端情形组合损失**:危机下相关性→1 的最大损失评估(压力 VaR),验证全局止损总闸生效。

## 16. 执行引擎(OMS/EMS)

### 16.1 OMS
订单生命周期状态机(新建→已提交→部分成交→全部成交/已撤/拒绝)· 幂等(`clientOrderId` UUIDv4)· 成交回报对账(本地 vs 交易所不一致告警)· 持仓恢复(重启拉真实持仓+审计重建)。

**多策略净额轧差(Netting)**〔G9 · 🟠〕:多策略并行可能自相残杀(A 买 BTC、B 卖 BTC = 内部对敲白付手续费)。订单网关做**内部净额轧差**(同标的反向先内部对冲再出净单)+ 跨策略同向限额 + 冲突仲裁。

### 16.2 EMS
算法拆单(TWAP/VWAP/POV/冰山)· 滑点控制(盘口深度感知,大单拆分,限价保护)· 智能路由(多所择优)· 撤补单(超时未成追价/撤单)· 市场冲击成本模型(回测与实盘同建模)。

### 16.3 适配器规范
API key 不硬编码(经 `infra/config.py`)· 密钥不打日志(`***` 脱敏)· 签名发送前校验 · 客户端限流(`exchange/rate_limiter.py`)· 适配器不调风控(职责分离)· 测试网/主网初始化校验。

### 16.4 交易成本分析(TCA)与策略容量〔G8 · 🟠〕
- **事后 TCA**:每笔/每日实施缺口(implementation shortfall)、与 VWAP/到达价基准对比、滑点归因;执行质量看板。
- **策略容量分析**:每策略容量曲线(资金规模 vs 预期收益衰减),超容量自动限制加仓。
- TCA 结果回流自进化(第 13 章),作为执行算法优化与权重调整依据。

### 16.5 账户/资金会计系统〔G5 · 🟠〕
> 不是"交易归因",是正式复式记账账本。
- **实时盯市(Mark-to-Market)**:已实现/未实现盈亏分离,逐笔精确。
- **多币种 NAV 合并**:加密 USDT 计价 + 美股 USD,经 FX 折算成统一基准币种净值。
- **复式记账 Ledger**:每笔资金流(成交/手续费/资金费率/分红/出入金)有借贷记录。
- **每日对账闭环**:账本 vs 交易所/券商对账单,差异告警。

### 16.6 税务处理〔G6 · 🟠 美股〕
- **Wash Sale 检测**:美股 30 天内回购亏损股,亏损不可抵税,自动标记。
- **Tax-Lot 会计**:FIFO/特定批次成本核算。
- **税务报表导出**:年度报表;加密按当地税务规则(如适用)。

# 第六部分 · 运行保障

## 21. 7×24 自动运行与自愈

### 21.1 调度器
策略时段控制(加密 7×24;美股按时段含盘前盘后自动启停)· 任务编排(行情订阅/因子计算/模型推理/AI 研报/对账/备份 cron)· 时区与节假日(美股交易日历 + 加密无休市)。

### 21.2 看门狗
监控全进程心跳 · 崩溃自动拉起(systemd/supervisor/K8s liveness)· 死锁检测(事件循环卡顿/行情停更/订单超时未回报 → 告警+重启)· **重启即恢复**(拉真实持仓 + 审计重建,SL/TP/入场价 key 全恢复)。

### 21.3 自愈策略

| 故障 | 自愈 |
|------|------|
| 行情断线 | 指数退避重连(≤60s)+ 快照对齐 + 缺口回补 |
| 交易所 API 异常 | 限流退避 + 熔断器半开探测 |
| DB 写锁争用 | 重试 + 连接池调优(教训:SQLite 单写者争用致登录失败,生产用 Postgres) |
| Redis 断连 | 本地缓冲 + 重连补发 |
| 策略异常 | 隔离该策略 + 告警,不影响其他 |
| 风控异常 | 立即熔断 + ERROR 告警(绝不静默) |

### 21.4 状态持久化
关键状态(持仓/挂单/策略内部状态)定期快照 + 事件溯源,任意时刻可重建;幂等恢复(不重复下单、不丢持仓、不错算盈亏)。

## 22. 监控、告警与可观测性

### 22.1 指标(Prometheus)

## 23. 安全与合规

### 23.1 密钥
API/JWT 密钥经 Vault/1Password 服务账户,不入代码不入仓库;启动校验缺失即拒启;疑似泄露立即轮换。

### 23.2 网络
交易所连接 TLS 1.2+ 无明文;内网 mTLS;安全响应头(HSTS 等)。

### 23.3 权限与审计
最小权限;敏感操作(改限额/紧急熔断/提现相关)双人复核 + 强审计;全操作不可篡改留痕定期归档。

### 23.4 资金安全红线
系统**不自动提现、不自动转账**(人工执行)· 交易所 API key 仅交易权限**禁用提现** · 测试网/主网物理隔离。

**加密钱包冷热分离**〔G4 · 🔴〕:交易所资金与自托管钱包分离,热钱包仅留交易所需最小额,余下冷存;充值/提币地址白名单(交易所侧);链上充值确认数监控 + 异常转账告警。

### 23.5 合规
交易记录与决策留痕满足内审回溯;美股/美期权遵守 IBKR 接口与监管时段;加密遵守交易所 API 条款与数据再分发限制。

### 25.5 运维成熟度 —— Runbook / On-Call / RTO-RPO / 变更管理〔G12 · 🟠〕
> 技术自愈 ≠ 组织就绪。补齐人的流程。
- **RTO/RPO 指标**:关键交易数据 **RTO < 5 分钟 / RPO < 1 秒**;明确分级目标。
- **On-Call 值班**:轮值 + 升级路径;P0 事故凌晨可触达,P0/P1 处置 runbook(手册)。
- **事故处置 Runbook**:常见故障(行情断/交易所故障/DB 故障/熔断)分步处置预案。
- **变更管理**:发布窗口 + **变更冻结**(CPI/FOMC/财报/到期日等高波动事件禁发布)+ 回滚预案。
- **事故复盘**:每次 P0/P1 复盘归档进机构记忆(接 7.3),喂回 LLM。
